# 计划和执行 plan_and_execute

![LangChain](https://pica.zhimg.com/50/v2-56e8bbb52aa271012541c1fe1ceb11a2_r.gif 'LangChain中文网')

计划和执行代理通过首先规划要做什么，然后执行子任务来实现目标。这个想法在很大程度上受到 [BabyAGI](https://github.com/yoheinakajima/babyagi) 和 ["Plan-and-Solve" paper](https://arxiv.org/abs/2305.04091) 的启发。

规划几乎总是由 LLM 完成。

执行通常由一个单独的代理（配备工具）完成。
