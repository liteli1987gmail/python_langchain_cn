# 欢迎贡献者

你好！感谢你对为LangChain做出贡献的兴趣。
作为一个在快速发展领域的开源项目，我们非常欢迎各种贡献，无论是新功能、改进的基础设施、更好的文档，还是错误修复。

## 🗺️ 指南

### 👩‍💻 贡献的方式

有许多方式可以为LangChain做出贡献。以下是一些常见的贡献方式：

- [**文档**](./documentation.mdx): 帮助我们改进文档，包括这一份！
- [**代码**](./code.mdx): 帮助我们编写代码，修复错误，或改进我们的基础设施。
- [**集成**](integrations.mdx): 帮助我们与你最喜欢的供应商和工具进行集成。
- [**讨论**](https://github.com/langchain-ai/langchain/discussions): 帮助回答使用问题，与用户讨论问题。

### 🚩 GitHub问题

我们的[问题](https://github.com/langchain-ai/langchain/issues)页面会及时更新错误，改进和功能请求。

我们有一套标签分类法，可以帮助你排序和发现感兴趣的问题。请使用这些标签来帮助组织问题。

如果你开始处理一个问题，请将其分配给自己。

如果你正在添加一个问题，请尽量将其聚焦在一个单一的、模块化的错误/改进/功能上。
如果两个问题有关联，或者阻塞，请将它们链接起来，而不是合并它们。

我们会尽量保持这些问题的最新状态，尽管
在这个领域的快速发展可能会使一些问题过时。
如果你注意到这种情况，请告诉我们。

### 💭 GitHub讨论

我们有一个[讨论](https://github.com/langchain-ai/langchain/discussions)页面，用户可以在那里提问，讨论设计决策，提出新的功能。

如果你能帮助回答问题，请尽量做到！这将使维护者能够花更多的时间专注于开发和错误修复。

### 🙋 获取帮助

我们的目标是拥有最简单的开发者设置。如果你在设置过程中遇到任何困难，请
联系维护者！我们不仅想帮助你解决问题，而且我们也希望确保这个过程
对未来的贡献者来说是顺畅的。

同样，我们在代码库中强制执行某些linting，格式化和文档标准。
如果你发现这些难以应对（或者只是觉得烦人），请随时联系维护者寻求帮助 -
我们不希望这些阻碍你将好的代码加入到代码库中。

# 🌟 认可

如果你的贡献已经被纳入到一个发布版本中，我们会想在Twitter上给你赞扬（只有你愿意的话）！
如果你有一个Twitter账户你希望我们提及，请在PR中告诉我们，或通过其他方式告诉我们。
